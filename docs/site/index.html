Why Use Capsule?

Capsule provides the operational tooling needed to run multi-agent workflows reliably in production environments.

Framework Agnostic

Write once in a declarative YAML manifest. Compile the configuration directly to production code for LangGraph, CrewAI, or OpenAI Agents.

Runtime Permissions

Limit what actions AI agents can take. Enforce read, write, or custom security boundaries. Request manual verification via interactive terminal prompts.

Signed Bundles

Package prompts, python tools, test parameters, and schema validations into a single signed .capsule archive verified by lockfile hash checkups.

Developer Quickstart

Get a secure agent workflow running in less than 2 minutes.

Bash 
# Install Capsule using pip
pip install capsule-agent

# Or run directly via uv tool execution (recommended)
uv tool run capsule --help
Bash 
# Initialize a new Capsule project scaffolding
capsule init my-agent-project

# Let's inspect the generated directory structure
cd my-agent-project
ls -R
Bash 
# Validate schema structure, schemas and edges
capsule validate .

# Scan local tool scripts for permission issues or security vulnerabilities
capsule scan .
Bash 
# Run local agent workflow with input payload
capsule run . --input examples/input-request.json

# Automatically pre-approve write operations in non-interactive environments
capsule run . --input examples/input-request.json --allow-permission write
Bash 
# Compile declarative spec to pure LangGraph code
capsule compile . --target langgraph

# Package project into a single redistributable .capsule file
capsule build .

Interactive YAML Spec Explorer

Click on any key in the capsule.yaml manifest below to understand its type, validation rules, and runtime impact.

capsule.yaml
name: translation-editor-agent
version: 0.1.0
description: Translates and edits text files.

input_schema:
  type: object
  required: [text]

models:
  default:
    provider: local
    model: gpt-4

agents:
  translator:
    prompt: agents/translator.md
    model: default
    tools: [write_file]

tools:
  write_file:
    type: python
    entrypoint: tools/write_file.py:write
    permission: write

workflow:
  start: translator
  steps:
    - id: translator
      type: agent
      agent: translator
      next: editor

permissions:
  tools:
    write_file: write

tests:
  - tests/test_translation.yaml
string Required

name

The unique identifier of the Capsule project. Used for compilation naming and bundle names.

Validation Rules

  • Must match alphanumeric characters and hyphens only.
  • Should not contain spaces.

Compiler / Runtime Impact

Determines the package name in compiled adapter projects, and the prefix for output bundles (e.g. my-agent-0.1.0.capsule).

💡 Pro Tip Keep names simple and descriptive to simplify CLI operations.

CLI Command Reference

Complete toolbelt for local development and workflow packaging.

Command Description Key Options
capsule init Initialize a blank Capsule directory structures. --name <str>, --overwrite
capsule validate Checks YAML schema conformances and routing cycles. --path <dir>
capsule scan Perform security code checks on python files. --path <dir>
capsule run Execute agent workflow locally with input payload. --input <file>, --allow-permission <perm>, --allow-all
capsule test Runs YAML workflow tests with mocking tools support. --path <dir>
capsule compile Compile manifest structure to specific SDK. --target <langgraph|crewai|openai_agents>
capsule build Build validated secure .capsule bundle. --output <dir>
capsule verify-bundle Verify bundle tamper status against lock file hashes. --bundle <file>
